# MCP API Spec: AI Phone Agent Platform

## Purpose
This document defines the v1 integration contract for:

- MCP runtime tools
- inbound external-agent bootstrap webhooks
- bidirectional WebSocket realtime session protocol
- conversation ownership behavior
- result and artifact retrieval

This spec is intended for engineers integrating external agent runtimes with the platform.

## Design Principles
- Keep the MCP surface small and high-level.
- Make internal platform ownership the default.
- Support external ownership for advanced integrations.
- Keep telephony, STT, TTS, transcript capture, recording, persistence, and observability inside the platform.
- Use webhook bootstrap plus WebSocket for inbound external handoff.
- Require deterministic runtime ownership and explicit fallback behavior.

## Core Concepts

### Conversation Ownership
Each call has two ownership values:

- `conversation_owner_requested`
- `conversation_owner_actual`

Allowed values:

- `internal`
- `external`

Default behavior:

- default conversation owner: `internal`
- inbound calls: `internal` unless inbound external handoff is enabled
- outbound calls: may use either `internal` or `external`

### Agent Profile
An agent profile defines:

- company identity
- tone and persona
- voice
- language
- prompt packs
- skill packs
- knowledge sources
- tool policies
- escalation rules

### Call vs AI Session
The platform stores:

- a telephony `call`
- an `ai_call_session`

The call tracks telephony/runtime facts.
The AI session tracks transcript, memory, summary, decisions, tool events, and conversation ownership behavior.

## Authentication

### MCP Authentication
MCP clients authenticate with a workspace API key.

Expected header:

```http
Authorization: Bearer mcp_<secret>
```

### Webhook Authentication
Inbound external bootstrap webhooks must be signed.

Recommended headers:

```http
X-Platform-Signature: <hmac_signature>
X-Platform-Timestamp: <unix_timestamp>
X-Platform-Event: inbound_call_requested
```

### WebSocket Authentication
External runtimes authenticate to the realtime session using a one-time session token generated by the platform.

Recommended connection form:

```text
wss://platform.example.com/realtime/external-call?session_token=<token>
```

## MCP Tool Surface

## Tool: `start_call`
Starts an outbound phone call.

### Intent
Use this tool when an external agent wants the platform to place a call and either:

- let the internal platform agent conduct the conversation
- let an external agent runtime conduct the conversation

### Input

```json
{
  "to": "+15551234567",
  "agent_profile_id": "ap_123",
  "goal": "Confirm tomorrow's appointment and answer basic questions.",
  "context": {
    "customer_name": "John Smith",
    "appointment_date": "2026-04-01",
    "appointment_window": "1 PM - 4 PM"
  },
  "language": "en",
  "conversation_owner": "internal",
  "outcome_schema": {
    "type": "object",
    "properties": {
      "confirmed": { "type": "boolean" },
      "needs_followup": { "type": "boolean" },
      "notes": { "type": "string" }
    }
  },
  "metadata": {
    "source": "chatgpt-agent",
    "campaign": "appointment-reminders"
  }
}
```

### Field Notes
- `to`: required destination phone number in E.164 format
- `agent_profile_id`: optional but recommended
- `goal`: required high-level objective
- `context`: optional structured context for the runtime
- `language`: optional preferred language
- `conversation_owner`: optional override, usually `internal` or `external`
- `outcome_schema`: optional structured expected result shape
- `metadata`: optional client-defined data persisted with the call

### Response

```json
{
  "call_id": "call_01",
  "session_id": "sess_01",
  "status": "initiated",
  "conversation_owner_requested": "internal",
  "conversation_owner_actual": "internal",
  "agent_profile_id": "ap_123",
  "created_at": "2026-03-30T20:10:00Z"
}
```

### Errors
- invalid phone number
- unauthorized MCP key
- missing or invalid agent profile
- requested ownership mode not allowed by workspace policy
- telephony provider unavailable

## Inbound External Handoff

## Feature Gate
Inbound external handoff only applies when:

- `allow_inbound_external_handoff = true`
- valid webhook URL is configured
- valid auth secret is configured

If not enabled, inbound calls go directly to the internal platform agent.

## Bootstrap Webhook
When an inbound call arrives and external handoff is enabled, the platform sends an HTTP webhook to the configured external orchestrator.

### Request

```http
POST /external-agent/inbound-call
Content-Type: application/json
X-Platform-Signature: <signature>
X-Platform-Timestamp: 1774900000
X-Platform-Event: inbound_call_requested
```

```json
{
  "event_type": "inbound_call_requested",
  "call_id": "call_02",
  "session_id": "sess_02",
  "workspace_id": "ws_01",
  "called_number": "+15557654321",
  "caller_number": "+15551234567",
  "received_at": "2026-03-30T20:30:00Z",
  "routing_context": {
    "twilio_number_id": "tn_01",
    "agent_profile_id": "ap_frontdesk",
    "language_hint": "en"
  },
  "memory_context": {
    "caller_known": true,
    "caller_name": "John Smith",
    "recent_summary": "Customer called about washer issue last week."
  },
  "session_token": "est_abc123",
  "ws_url": "wss://platform.example.com/realtime/external-call",
  "reply_deadline": "2026-03-30T20:30:08Z"
}
```

### Expected Response
The bootstrap webhook only acknowledges receipt. It does not carry the conversation.

```json
{
  "accepted": true
}
```

If the orchestrator cannot handle the request, it should still return deterministically:

```json
{
  "accepted": false,
  "reason": "no_runtime_available"
}
```

The platform should then continue with fallback behavior.

## Readiness Timeout
The external runtime must become ready before `external_ready_timeout_ms`.

Recommended default:

- `8000 ms`

If the external runtime is not ready in time:

- `conversation_owner_requested = external`
- `conversation_owner_actual = internal`
- `fallback_reason = external_timeout`

The platform continues the inbound call using the internal agent.

## WebSocket Realtime Protocol

## Connection
The external runtime connects to the `ws_url` using the one-time `session_token`.

Recommended query string:

```text
wss://platform.example.com/realtime/external-call?session_token=est_abc123
```

## Lifecycle
1. Platform sends session start event.
2. External runtime must send `agent_ready`.
3. Platform streams transcript and call events.
4. External runtime sends `reply_text` events and optional actions.
5. Platform performs TTS and plays synthesized speech into the call.
6. Platform persists recordings, transcript, and events.

## Client-to-Server Events

### `agent_ready`
Must be sent before the readiness deadline.

```json
{
  "type": "agent_ready",
  "session_id": "sess_02",
  "agent_name": "External Concierge",
  "capabilities": {
    "actions": true,
    "controls": true
  }
}
```

### `reply_text`
Primary response event in v1.

```json
{
  "type": "reply_text",
  "text": "Of course. I can help with that. What day would work better for you?",
  "end_of_turn": true
}
```

### `action`
Optional structured action emitted by the external agent.

```json
{
  "type": "action",
  "name": "mark_intent",
  "payload": {
    "intent": "reschedule_appointment"
  }
}
```

### `control`
Optional control command.

```json
{
  "type": "control",
  "name": "hangup"
}
```

Allowed control names in v1:

- `hangup`
- `transfer_to_human`
- `mark_goal_complete`

### `heartbeat`

```json
{
  "type": "heartbeat",
  "ts": "2026-03-30T20:30:10Z"
}
```

## Conversation Ownership Rules

## Workspace Defaults
Workspace runtime settings define:

- `conversation_owner_default`
- `allow_inbound_external_handoff`
- `external_ready_timeout_ms`
- `inbound_fallback_mode`

## Effective Rules
- If outbound call does not specify `conversation_owner`, use workspace default.
- If inbound call arrives and inbound external handoff is disabled, force `internal`.
- If inbound external handoff is enabled, request `external`, but actual owner becomes `external` only after `agent_ready`.
- If readiness times out, set actual owner to `internal`.

## Artifact and Event Persistence
The platform persists artifacts in both ownership modes:

- recording
- transcript
- summary
- extracted facts
- QA evaluation
- memory update

The platform also persists:

- requested owner
- actual owner
- bootstrap status
- fallback reason
- external runtime metadata

## Status Model
Suggested call/session statuses:

- `initiated`
- `ringing`
- `in_progress`
- `completed`
- `failed`
- `canceled`
- `fallback_to_internal`

Suggested external bootstrap statuses:

- `not_requested`
- `requested`
- `accepted`
- `runtime_connecting`
- `ready`
- `timed_out`
- `failed`

## Error Cases

### MCP Layer
- invalid API key
- invalid tool input
- ownership mode forbidden by workspace policy
- invalid or unknown call ID

### Inbound Bootstrap
- webhook timeout
- bad webhook response
- invalid signature verification state
- no runtime available

### WebSocket Layer
- invalid session token
- expired token
- duplicate connection
- missing `agent_ready`
- heartbeat timeout

### Realtime Voice Layer
- STT provider unavailable
- TTS provider unavailable
- telephony disconnect
- transcript stream interrupted

## Recommended Implementation Defaults
- `conversation_owner_default = internal`
- `allow_inbound_external_handoff = false`
- `external_ready_timeout_ms = 8000`
- `inbound_fallback_mode = fallback_to_internal`
- external reply mode = `text + optional actions`
- platform-owned TTS in both ownership modes

## Compatibility Notes
- Keep MCP tool contracts stable and high-level.
- Avoid exposing low-level telephony or media primitives over MCP in v1.
- External inbound handoff should be implemented as a separate realtime integration contract, not as an MCP tool.
- Future versions may add:
  - per-call ownership override policy
  - per-profile ownership defaults
  - richer action schemas
  - mid-call ownership failover
  - external audio reply mode
