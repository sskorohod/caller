# Caller — Полный обзор приложения

## Что такое Caller

Caller — это B2B SaaS-платформа для создания AI-телефонных агентов и живого перевода звонков. Платформа позволяет бизнесам автоматизировать телефонные коммуникации с помощью искусственного интеллекта: входящие и исходящие звонки, транскрипция, запись, аналитика, а также живой перевод разговоров в реальном времени.

### Ключевые возможности

- **AI Phone Agents** — создание виртуальных телефонных агентов с настраиваемыми голосами, промптами и навыками
- **Живой переводчик** — перевод телефонных разговоров в реальном времени на 10+ языков
- **Миссии** — AI-ассистент для планирования и выполнения звонков через чат-интерфейс
- **MCP Server** — API-доступ для интеграции с внешними системами (ChatGPT, Claude Desktop и др.)
- **BYOK (Bring Your Own Keys)** — возможность использовать собственные API-ключи провайдеров

## Модель оплаты

### Депозит
Платформа работает по модели предоплаченного депозита. Пользователь пополняет баланс через Stripe, и стоимость использования провайдеров платформы списывается из баланса автоматически.

### Свои ключи (BYOK)
Пользователи могут подключить собственные API-ключи для любого провайдера. При использовании собственных ключей плата за этот провайдер не взимается. Можно комбинировать: например, свой Twilio + AI-провайдер платформы.

### Пробный период
Планы Agents и Agents + MCP предоставляют 15-дневный бесплатный пробный период. Карта не требуется для активации. По истечении пробного периода нужно оформить подписку для продолжения использования.

## Модули платформы

### 1. Обзор (Dashboard Overview)

Главная страница дашборда с ключевыми метриками:
- Количество звонков (сегодня, за неделю, всего)
- Процент успешных звонков
- Активные звонки в реальном времени
- Средняя длительность звонка
- Распределение по статусам и направлениям
- Разбивка тональности (позитивный/нейтральный/негативный)
- Средняя оценка качества
- Затраты по компонентам (LLM, TTS, STT, телефония)

### 2. AI-агенты

#### Создание агента
Каждый AI-агент настраивается с помощью следующих параметров:

**Основные поля:**
- Имя (machine-readable) и отображаемое имя
- Описание
- Название компании и идентичность компании
- Аватар (выбор из 8 стандартных или загрузка своего)
- Язык (en, ru, es, de, fr, auto)

**Настройка голоса:**
- Провайдер: xAI Grok TTS (основной), OpenAI TTS, ElevenLabs
- Выбор голоса (зависит от провайдера)
- Настройки голоса (JSON)

**Настройка LLM:**
- Провайдер: Anthropic, OpenAI, xAI
- Модель:
  - Anthropic: Claude Sonnet 4.5, Claude Opus 4.5, Claude Haiku 3.5
  - OpenAI: GPT-4.1, GPT-4.1 Mini, GPT-4.1 Nano, o3, o4-mini, GPT-4o
  - xAI: Grok 3, Grok 3 Mini Fast
- Температура (0–1)

**STT (распознавание речи):**
- Провайдер: Deepgram (основной) или OpenAI Whisper

**Поведение:**
- Системный промпт
- Приветственное сообщение
- Бизнес-режим и теги
- Правила эскалации
- Политики инструментов
- Память (вкл/выкл, период ретроспективы)

#### Промпт-паки
Промпт-паки — это переиспользуемые блоки системных инструкций, которые можно прикреплять к агентам с приоритетом. Каждый промпт-пак содержит:
- Название и описание
- Содержимое (текст промпта)
- Категорию
- Версию

Один агент может иметь несколько промпт-паков с разными приоритетами.

#### Скилл-паки
Скилл-паки определяют конкретные навыки и сценарии поведения агента:
- Намерение (intent) — что скилл делает
- Правила активации — когда скилл срабатывает
- Необходимые данные
- Последовательность инструментов
- Условия эскалации
- Критерии завершения
- Правила прерывания
- Правила диалога

AI может рекомендовать подходящие скилл-паки на основе конфигурации агента.

#### База знаний
Базы знаний позволяют агентам работать с информацией о компании:
- Создание баз знаний с названием и описанием
- Добавление документов разных типов: документ, FAQ, политика, прайс, устранение неполадок
- Семантический поиск через векторные эмбеддинги (pgvector)
- AI-ассистент для форматирования контента
- Привязка баз знаний к агентам

### 3. Звонки

#### Исходящие звонки
- Создание через API/MCP: номер, агент, цель звонка, контекст
- Создание через дашборд (миссии)
- Идемпотентность через Idempotency-Key (24-часовое окно)
- Telegram-уведомления с ссылкой на мониторинг

#### Входящие звонки
- Настройка через телефонные подключения
- AI-автоответ с назначенным агентом
- Настраиваемая задержка автоответа
- Поддержка внешнего владельца диалога (external conversation owner)

#### Диалер
- Ручной набор номера из браузера
- Настройка языка STT
- Режим голосового перевода
- Выбор провайдера TTS

#### Запись и транскрипция
- Автоматическая запись всех звонков
- Хранение в MinIO (внутреннее) или Twilio
- Полная транскрипция диалога
- AI-генерированное резюме
- Извлечение ключевых фактов и действий
- Анализ тональности
- Оценка качества (0–10)

#### Статистика звонков
- 30-дневная скользящая статистика
- Разбивка по статусам, направлениям, тональности
- Затраты по компонентам
- Топ-5 агентов по количеству звонков
- Ежедневный график звонков

### 4. Живой переводчик

B2C-сервис для перевода телефонных разговоров в реальном времени.

**Как это работает:**
1. Подписчик звонит на выделенный номер
2. Система подключается к звонку как конференция (merge)
3. Речь собеседника распознаётся (Deepgram STT)
4. Текст переводится через LLM
5. Перевод озвучивается (xAI Grok TTS)
6. Подписчик слышит перевод в реальном времени

**Настройки:**
- Пара языков (мой язык ↔ целевой язык)
- Режим: голос, текст или оба
- Кто слышит перевод: подписчик или оба
- Направление: двунаправленный или однонаправленный
- Тон: нейтральный, деловой, дружеский, медицинский, юридический
- Персональный контекст (до 2000 символов)
- Приветственное сообщение
- Выбор голоса TTS

**Умные функции:**
- Спекулятивный перевод промежуточных результатов (снижает воспринимаемую задержку)
- Удаление слов-паразитов
- Контекстное сохранение для точности
- Мониторинг в реальном времени через дашборд

### 5. Миссии

Миссии — это AI-ассистент для планирования и выполнения звонков через чат-интерфейс.

**Процесс:**
1. Создание новой миссии (статус: черновик)
2. Общение с AI в чате — описание задачи
3. AI задаёт уточняющие вопросы, собирает информацию:
   - Номер телефона для звонка
   - Цель звонка
   - Язык
   - Контекст (имя собеседника, компания и т.д.)
4. AI формирует план звонка (JSON)
5. Пользователь подтверждает — миссия переходит в статус «готова»
6. Нажатие «Выполнить» — инициирует звонок
7. После завершения — результат и резюме

**Автоматические функции:**
- Распознавание номеров телефона в тексте чата
- Загрузка истории звонков с найденным номером
- Подгрузка памяти абонента (факты из прошлых звонков)
- Повторные попытки (до 3 по умолчанию)

**Статусы миссии:**
draft → ready → scheduled → calling → in_progress → on_hold → completed / failed / cancelled

### 6. Телефонные номера

Управление телефонными номерами через интеграцию с Twilio.

**Процесс подключения:**
1. Указать учётные данные Twilio (Account SID + Auth Token)
2. Система загружает список доступных номеров из Twilio
3. Активировать нужный номер (создать подключение)
4. Настроить:
   - Входящие звонки (вкл/выкл)
   - Исходящие звонки (вкл/выкл)
   - AI-автоответ (вкл/выкл)
   - Агент по умолчанию для входящих
   - Дружественное имя

### 7. Настройка провайдеров (BYOK)

Поддерживаемые провайдеры:

| Провайдер | Тип | Что нужно | Где получить |
|-----------|-----|-----------|-------------|
| **Twilio** | Телефония | Account SID + Auth Token | console.twilio.com |
| **Anthropic** | LLM | API Key | console.anthropic.com |
| **OpenAI** | LLM/TTS/STT | API Key | platform.openai.com |
| **Deepgram** | STT | API Key | console.deepgram.com |
| **xAI** | LLM/TTS | API Key | console.x.ai |
| **ElevenLabs** | TTS (опц.) | API Key | elevenlabs.io |
| **Telegram** | Уведомления | Bot Token | @BotFather в Telegram |

Для каждого провайдера можно переключаться между режимами:
- **Свои ключи (Own)** — используются ваши API-ключи, плата не взимается
- **Платформа (Platform)** — используются ключи платформы, стоимость списывается из депозита

Все учётные данные шифруются AES-256-GCM при хранении.

### 8. Интеграции

#### API-ключи
- Создание API-ключей для программного доступа
- Формат: `mcp_` + случайная строка
- Ключ показывается только один раз при создании
- Хранение через SHA-256 хэш
- Возможность отзыва (revoke)

#### Вебхуки
- Настройка HTTPS-эндпоинтов для получения событий
- Поддерживаемые события:
  - `call.started` — звонок начался
  - `call.completed` — звонок завершён
  - `call.failed` — звонок не удался
  - `session.summary_ready` — резюме готово
- Тестирование через отправку пробного события
- Опциональный секретный ключ для верификации

#### Коннекторы данных
- Создание коннекторов к внешним системам
- Настраиваемый тип и конфигурация
- Тестирование подключения
- Выполнение действий

#### OAuth 2.0
- Создание OAuth-приложений (Client ID + Client Secret)
- Авторизация по коду (Authorization Code Flow)
- Поддержка redirect URI
- Токены доступа со сроком действия 90 дней
- Интеграция с ChatGPT Actions, Claude Desktop и др.

#### MCP Server
- Программный доступ ко всем функциям платформы
- Инструменты для звонков, агентов, знаний, миссий, биллинга
- Подключение через npx или прямую установку
- Аутентификация через API-ключ

### 9. Настройки рабочего пространства

#### Общие
- Название рабочего пространства
- Имя владельца
- Телефонные номера (до 3, E.164 формат)
- Часовой пояс
- Владелец диалога по умолчанию (internal/external)

#### Внешний вид
- Тема оформления
- Акцентный цвет

#### Комплаенс
- Объявление о записи звонка (вкл/выкл)
- Объявление об AI-агенте (вкл/выкл)
- Задержка автоответа на входящие (5–120 сек)

#### Команда
- Приглашение участников по email
- Роли: Owner, Admin, Operator, Analyst
- Управление доступом

### 10. Аудит

Система аудита отслеживает все важные действия:
- Изменение учётных данных провайдеров
- Создание/отзыв API-ключей
- Изменение вебхуков и коннекторов
- Управление подпиской
- IP-адрес и пользователь для каждого действия

## Безопасность

- **AES-256-GCM** — шифрование учётных данных провайдеров
- **SHA-256** — хэширование API-ключей
- **Timing-safe сравнение** — защита от атак по времени
- **HTTPS** — обязательный для вебхуков
- **RBAC** — ролевая модель доступа (Owner, Admin, Operator, Analyst)
- **Изоляция рабочих пространств** — все запросы скопированы workspace_id
- **Верификация учётных данных** — Twilio проверяется при сохранении
- **Аудит** — все изменения конфигурации логируются

## Архитектурные особенности

### Dual-Auth система
- **JWT** — аутентификация через браузер (дашборд)
- **API Key** — аутентификация для MCP/программного доступа
- Многие эндпоинты поддерживают оба метода

### Real-time архитектура
- **Socket.IO** — обновления миссий, живая транскрипция, перевод
- Сервер отправляет статусы, фронтенд подписывается на обновления

### Гибкость провайдеров
- Несколько провайдеров для каждой задачи (LLM, TTS, STT)
- Цепочка fallback: основной → резервный → дефолтный
- Выбор модели зависит от задачи

### Идемпотентность
- Создание звонков поддерживает Idempotency-Key (24-часовое окно)
- Миссии имеют логику повторных попыток
- Атомарные batch-операции для скилл-паков и промпт-паков

### Память абонентов
- Профили абонентов привязаны к номерам телефонов
- Факты из прошлых звонков сохраняются для контекста
- Миссии автоматически загружают историю
